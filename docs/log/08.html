GETH + NODE + WEB3

To initiate a geth node (I'm running Geth 1.8.20-stable built on amd64 with go1.11.4): Since Web3 v1.00 is only provided as an npm package, we will later need to use webpack to transpile it into native js, for now we'll just invoke node from the command line Web3 1.00 has depreciated http providers, so we now need to use IPC to communicate with the geth node: 
geth --datadir /tmp/geth --networkid 15 --rpc --rpcport "8545" --rpcaddr "localhost" --rpccorsdomain "*"
then in node we can run: 
const net = require('net');
const Web3 = require('web3');
const web3 = new Web3('/tmp/geth/geth.ipc', net);
web3.eth.getAccounts().then(console.log); //outputs accounts array
to connect via remix, we can set Environment->web3 provider->http://localhost:8545 (in the Run tab), so that we can deploy a smart contract to our node.

A Note on Webpack

Since we now need to use webpack to provide the Web3 library, we need a workaround to get button onclick="myfunc();"> to work since the function won't be added to the global scope when we transpile w/ webpack As suggested in the SO answer, we could assign it to the window object: window.myFunc = function(){...} Or use js to detect clicks document.getElementById('buttonID').onClick = function(){...} N.B. For the second method to work, we need to embed the script after the button is created in the DOM (ie. at the bottom of the html body)
